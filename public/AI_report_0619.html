详细技术说明

Resources

将数据暴露为只读资源，支持文件、数据库记录、API 响应等

Prompts

提供可重用的提示模板和工作流，方便用户选择和复用

Tools

提供可由 LLM 自动调用的动作，支持 JSON Schema 描述

Sampling

允许 Server 反向请求 Client 进行模型采样，支持人在回路

Roots

告知 Server 需要关注的 URI 范围，作为工作边界

Transports

实现可插拔的通信机制，支持 stdio 和 HTTP 等传输

公共基础设施

  • • 巨额资本支出（相当于建设电网）
  • • 通过同质化API提供智能服务
  • • 按量计费（每百万tokens收费）
  • • 对稳定性要求极高

晶圆厂特性

  • • 深度技术树研发与技术机密
  • • 4纳米工艺 ≈ 10²⁰次浮点运算集群
  • • 使用英伟达GPU = 无晶圆厂
  • • 谷歌TPU = 拥有晶圆厂

操作系统

  • • 复杂软件生态系统，非简单商品
  • • 系统/用户空间（提示空间 ≈ 内核/用户空间）
  • • 闭源供应商（GPT/Claude ≈ Mac/Windows）
  • • 开源生态（Llama ≈ Linux）

分时大型机

  • • 类似早期分时操作系统

  • • 文本聊天 ≈ 终端界面
  • • 消费者→企业→政府的逆向普及

LLM 心理学：高智能与认知缺陷并存

根本问题

应对策略
将LLM用于擅长的任务，注意锯齿状边缘，让人工参与其中
